# @docmux/wasm npm Package — Design Spec

> Date: 2026-03-24
> Status: Approved
> Scope: `npm/` directory + `crates/docmux-wasm` changes

## Goal

Publish `@docmux/wasm` to npm — a universal (browser + Node) WASM package that exposes docmux's document conversion capabilities to JavaScript/TypeScript consumers.

## Design Decisions

1. **Universal target** — works in browsers (via bundlers like webpack/vite) and Node.js.
2. **`wasm-pack`** as the build tool — generates JS glue, `.d.ts` types, `.wasm` binary, and package scaffolding automatically.
3. **Thin wrapper** — no additional JS/TS abstraction. Re-export the 4 WASM functions directly with auto-generated types.
4. **Add `LatexReader`** to the WASM crate registry — enables `convert(input, "latex", "html")` from JS.

## WASM Crate Changes

### `crates/docmux-wasm/Cargo.toml`

Add dependency:

```toml
docmux-reader-latex = { workspace = true }
```

### `crates/docmux-wasm/src/lib.rs`

Add import and register `LatexReader` in the registry:

```rust
use docmux_reader_latex::LatexReader;

// Inside build_registry() or wherever the registry is constructed:
reg.add_reader(Box::new(LatexReader::new()));
```

No API changes — same 4 exported functions: `convert`, `markdownToHtml`, `inputFormats`, `outputFormats`. But `inputFormats()` now returns `["markdown", "latex"]` and `convert(input, "latex", "html")` works.

## npm Package Structure

```
npm/
├── package.json          (publishable @docmux/wasm)
├── build.sh              (wasm-pack build orchestrator)
├── test.mjs              (Node.js smoke test)
└── pkg/                  (generated by wasm-pack, gitignored)
    ├── docmux_wasm_bg.wasm
    ├── docmux_wasm.js
    ├── docmux_wasm.d.ts
    └── package.json      (generated, overridden by parent)
```

### `npm/package.json`

```json
{
  "name": "@docmux/wasm",
  "version": "0.1.0",
  "description": "Universal document converter — WASM bindings",
  "license": "MIT",
  "repository": {
    "type": "git",
    "url": "https://github.com/aguschirico/docmux"
  },
  "type": "module",
  "main": "./pkg/docmux_wasm.js",
  "types": "./pkg/docmux_wasm.d.ts",
  "files": ["pkg/"],
  "scripts": {
    "build": "./build.sh",
    "prepublishOnly": "npm run build"
  },
  "sideEffects": ["./pkg/docmux_wasm.js"]
}
```

- `type: "module"` — ESM output
- `files: ["pkg/"]` — only publish the generated output
- `sideEffects` — prevents bundlers from tree-shaking the WASM init glue

### `npm/build.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
wasm-pack build crates/docmux-wasm --target bundler --out-dir ../../npm/pkg
```

- `--target bundler` — ESM output compatible with webpack, vite, parcel, etc.
- `--out-dir ../../npm/pkg` — output directly into the npm package

### `npm/test.mjs`

Note: `--target bundler` output uses `import.meta.url`-based WASM loading designed for bundlers. To run in Node.js, we read the `.wasm` file manually and pass it to `init()`.

```js
import { readFile } from 'node:fs/promises';
import { strict as assert } from 'node:assert';
import init, { convert, markdownToHtml, inputFormats, outputFormats } from './pkg/docmux_wasm.js';

// Load WASM bytes manually (bundler target doesn't auto-resolve in Node)
const wasmBytes = await readFile(new URL('./pkg/docmux_wasm_bg.wasm', import.meta.url));
await init(wasmBytes);

// Test markdown to HTML
const html = markdownToHtml("# Hello\n\nWorld");
assert(html.includes("<h1>"), "markdownToHtml should produce h1");

// Test input formats include latex
const inputs = inputFormats();
assert(inputs.includes("latex"), "inputFormats should include latex");

// Test latex to HTML
const texHtml = convert("\\textbf{bold}", "latex", "html");
assert(texHtml.includes("<strong>"), "convert latex to html should work");

// Test output formats
const outputs = outputFormats();
assert(outputs.includes("html"), "outputFormats should include html");
assert(outputs.includes("latex"), "outputFormats should include latex");

console.log("all tests passed");
```

Run with: `node npm/test.mjs`

## .gitignore

Add `npm/pkg/` to root `.gitignore`. Note: the existing `.gitignore` already has `pkg/` and `*.wasm` glob rules, but adding the explicit path improves clarity for contributors.

## CI Changes

Replace the `wasm` job in `.github/workflows/ci.yml`:

```yaml
  wasm:
    name: WASM Build & Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
        with:
          targets: wasm32-unknown-unknown
      - uses: Swatinem/rust-cache@v2
      - name: Install wasm-pack
        run: curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh
      - name: Build WASM package
        run: wasm-pack build crates/docmux-wasm --target bundler --out-dir ../../npm/pkg
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - name: Run smoke test
        run: node npm/test.mjs
```
